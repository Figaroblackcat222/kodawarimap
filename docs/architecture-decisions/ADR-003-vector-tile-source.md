# ADR-003: ベクタータイル供給元（Protomaps PMTiles 自己ホスト）

## 状況
地図アプリの心臓部。要件「エリア保存（タイルキャッシュ）」と核心価値「オフライン」「プライバシー」に直結する。

## 最重要制約
多くの商用タイルプロバイダは ToS でオフラインキャッシュ／一括ダウンロードを禁止している。これが選択を大きく絞る。

## 選択肢
- Protomaps（PMTiles 自己ホスト）
- MapTiler（商用API）— 無料枠でオフラインキャッシュ実質禁止
- OpenFreeMap
- 自前タイル生成 — 運用コスト大

## 決定
**Protomaps（PMTiles 自己ホスト）を採用。**
**国土地理院タイルは登山カテゴリ向けに将来重畳（fast-follow）。**

## 理由
- オフラインキャッシュ権利が完全に自由（OSSデータ）
- PMTilesは惑星全体が1ファイル。静的配信のみでタイルサーバー不要 → [ADR-002](ADR-002-no-backend-mvp.md) と完全整合
- 第三者APIにユーザーの閲覧位置が流れない → 核心価値「プライバシー」に合致
- 動的スタイル切替は「同じタイル＋styleJSON差し替え」で実現でき要件と一致

## トレードオフ
- OSSベースのため日本の登山地形（等高線）が弱い → 登山カテゴリは将来 国土地理院タイルを重畳して補う
- `MapLibreStyleAdapter` が「カテゴリ → {タイルソース, styleJSON}」を解決する設計なので、後から地理院を足してもDomain/UseCaseは無傷
