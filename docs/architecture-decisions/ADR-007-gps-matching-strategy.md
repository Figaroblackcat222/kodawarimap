# ADR-007: GPS マッチング戦略（カテゴリー別閾値＋同日判定）

## 状況

写真をアップロードした際に「同じ場所の写真か」を判定する仕組みが必要。
当初は GPS 座標の完全一致で判定していたが、スマートフォンの GPS 誤差（屋外3〜10m、屋内10〜30m）により実運用でほぼ発動しないことが判明した。

また「ピンが何を表すか」の設計上の問題もあった。

- **誤って分離**（同じ場所なのに別ピン）: 地図が散らかり、毎回起きる → 体験として致命的
- **誤って合算**（別の場所なのに同じピン）: 手動で分割すれば直せる → 許容できる

## 選択肢

1. **完全一致**（旧実装）: GPS が完全一致の場合のみ既存ピンに追加。実用上ほぼ発動しない
2. **固定距離閾値**（採用）: 半径 N メートル以内を「同じ場所候補」とする
3. **GPSHPositioningError による動的閾値**: EXIF の GPS 精度情報を使って閾値を動的に決定。iOS 専用タグのため Android では機能しない

## 決定

**カテゴリー別 Haversine 距離閾値 ＋ 同日（takenAt）判定**

```
同じ場所 = distanceMeters(pinA, pinB) <= threshold[category]
同じ体験 = 同じ場所 AND isSameDay(pinA.takenAt, pinB.takenAt)
```

| カテゴリー                                   | 閾値 | 理由                                   |
| -------------------------------------------- | ---- | -------------------------------------- |
| 汎用・旅行・テーマパーク・海ビーチ・公園自然 | 50m  | 屋内GPS誤差（Wi-Fi測位）を吸収         |
| 食事                                         | 5m   | 同じビル内の別店舗を誤マージしない     |
| 登山・キャンプ                               | 100m | 山の稜線・山頂付近のGPS誤差が大きい    |
| 釣り・神社仏閣・温泉                         | 30m  | 同一ポイント・スポットの範囲として適切 |

| 条件                     | 動作                                                                                       |
| ------------------------ | ------------------------------------------------------------------------------------------ |
| 閾値以内 ＋ 同日         | マージ確認バー表示 →「追加する」で既存ピンに写真追加 /「新規ピンを作成」で仮置きモードへ   |
| 閾値以内 ＋ 別の日       | 仮置きモード（タイトル・カテゴリーは最近傍ピンから継承）                                   |
| 閾値以内 ＋ takenAt なし | マージ確認バー表示 →「追加する」で最近傍ピンに写真追加 /「新規ピンを作成」で仮置きモードへ |
| 閾値超                   | 仮置きモード（ファイル名タイトル）                                                         |

## 理由

- **カテゴリー別閾値**: 食事（飲食店）は同ビル内の別店舗を誤マージしやすいため5mに絞る。登山は山頂付近のGPS精度が低いため100mに広げる
- **同日判定**: 同じ場所への再訪は「別の体験」として記録したい（直近の体験だけ絞り込む検索ユースケースへの対応）
- **タイトル継承**: 再訪ピンに場所名を手入力する手間を省く。「こだわりの場所」として同名で蓄積される
- **距離閾値固定**: GPSHPositioningError は iOS 専用で Android では取得できないため、全端末で一貫した挙動を優先

## GPS位置調整（仮置きモード）

GPS誤差が大きい場合や意図的に別の座標にしたい場合のために、ユーザーが手動で位置を微調整できる仮置きモードを実装した。

```
写真追加 → Exif GPS座標に琥珀色ドラッグ可能マーカーを仮置き
  └─ 「ここに決定」→ マーカーの現在座標で新規ピン保存（マージ判定なし）
  └─ 「キャンセル」→ 仮置きマーカー破棄
```

- 仮置き中はDBへの書き込みが発生しない（確定まで安全）
- マージ確認で「新規ピンを作成」を選んだ場合も同じ仮置きモードに遷移
- 仮置きモードで確定した座標にはマージ判定を適用しない（ユーザーが意図的に指定した位置として扱う）

## トレードオフ

- 食事5mは屋内GPS誤差（10〜30m）より小さいため、同一店舗でも仮置きモードに進みやすい。ユーザーがドラッグ不要と判断すれば「ここに決定」をそのまま押せばよい
- 安全弁として**写真分割機能**（ピン詳細画面から1枚の写真を新規ピンに移動）も引き続き提供

## 将来の検討事項

- `GPSHPositioningError` が取得できる場合（主にiOS）は動的閾値を使う拡張
- 設定画面からユーザーが閾値を変更できるオプション
