# ADR-002: MVPはバックエンドなし

## 状況

MVP要件は「ログイン不要・ローカル完結」。バックエンドを今用意すべきか判断する必要がある。

## 選択肢

- バックエンドなし（純クライアントサイド）
- 軽量BFFを最初から用意

## 決定

**バックエンドなし。MVPは100%クライアントサイドで完結。**

## 理由

- MVP要件にサーバーを必要とする機能がない（全データ端末内、認証不要）
- 今作ると要件のない機能を運用することになる（YAGNI）
- 開発速度・コストの両面で最速・最安
- Clean Architecture により、将来のクラウド同期は `port` の新実装（`CloudPinRepository`）を足すだけで Domain/UseCase は無傷

## 将来

クラウドバックアップ機能が具体化したときに初めてバックエンドを導入する。Web公開（プロ版）の認証＋DBもその際の決定事項。
